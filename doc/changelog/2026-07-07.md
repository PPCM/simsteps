# Changelog - 2026-07-07

## Summary
Brick 4 of the non-technician import plan: assisted recalibration. A new engine KPI (lines/hour/operator) feeds a « Recalage » section in the Indicateurs window — the user types the productivity observed in their WMS, the app bisects the per-line pick time over full deterministic runs until the simulation reproduces it (±5 %), and « Appliquer » writes the found value into the scenario overrides. Degenerate cases are diagnosed in French: unreachable target, and flat response (demand-limited system where the pick time doesn't drive productivity).

## Notes
The flat-response guard exists because the first UI test run exposed it: on the default demo scenario the operators are demand-limited, so any pick time "matched" the observed productivity and the bisection converged anywhere. Calibration now requires a load-sensitive configuration and says so.
