# Changelog - 2026-07-04

## Summary
All 7 steps of the delivery plan: standalone discrete-event simulation engine with unit tests and console KPI export; REST API + PostgreSQL (migrations, seed, CRUD, run recording) with a working docker-compose stack; static 3D render of the warehouse (Three.js, orbit camera); operator animation driven by the engine (in-browser simulation, recorded timeline, interpolated playback at x1/x10/x60); full UI panel with live KPIs, spaghetti trails, floor heatmap, run saving and comparison mode; Helm chart validated with lint and template; French README and end-to-end Docker verification from a clean volume.

## Changes

- [package.json] Created: package `simsteps` (version 0.1.2 after two releases), ES modules, license Apache-2.0 (aligned with the repository LICENSE), engines node >=22, scripts `npm test` (Node native runner), `npm run sim` (CLI), `npm run dev`/`start` (server); dependencies fastify, @fastify/static, @fastify/rate-limit, pg, three
- [sim/rng.js] Created: seeded deterministic PRNG (mulberry32) with randInt / randExponential / randChoice helpers
- [sim/eventQueue.js] Created: stable binary min-heap event queue ordered by (time, insertion order)
- [sim/graph.js] Created: circulation graph (positioned nodes, directed/bidirectional edges) with A* shortest path
- [sim/warehouse.js] Created: warehouse JSON loader building the circulation graph (bay nodes per aisle, front/back corridors, facilities) and the picking-slot map
- [sim/orders.js] Created: B2C (1-3 lines, qty 1-3) and B2B (10-50 lines, qty 5-20, client pool) order generation
- [sim/strategies.js] Created: pluggable picking strategies — orderByOrder (one mission per order, FIFO) and zoneWave (zone-grouped waves capped at waveSize)
- [sim/engine.js] Created: discrete-event engine — Poisson order arrivals, nearest-idle-operator assignment, serpentine pick routes, B2C drops at nearest workshop / B2B at shipping, busy-time and distance tracking, per-edge traffic aggregation exposed as `traffic` in the result; rendering hooks onEvent, onTravel (full path + duration at each departure) and onState (moving/picking/dropping/idle transitions)
- [sim/kpi.js] Created: KPI computation (orders/h, lines/h, avg distance, occupancy, cycle time, pending) and French console formatting
- [sim/cli.js] Created: runs a scenario JSON against a warehouse JSON and prints KPIs
- [data/warehouse-example.json] Created: example warehouse — 6 aisles, 12 racks, 204 picking slots, 2 workshops, shipping and receiving zones
- [data/scenario-example.json] Created: baseline scenario (orderByOrder, 5 operators, 30 orders/h, 70% B2C)
- [data/scenario-waves.json] Created: comparison scenario using the zoneWave strategy
- [tests/unit/sim/*.test.js] Created: tests covering rng, eventQueue, graph/A*, warehouse construction and reachability, order generation, strategies, engine end-to-end (determinism, KPI coherence, traffic invariant) and KPI computation/formatting
- [db/migrations/001_init.sql] Created: warehouses, scenarios, runs tables (JSONB documents, scenario snapshot + KPIs + aggregated traffic per run)
- [db/migrate.js] Created: versioned SQL migration runner, transactional, applied automatically at startup
- [db/seed.js] Created: inserts example warehouse and the two example scenarios on first start only
- [server/db.js] Created: pg pool creation and wait-for-database retry loop
- [server/validate.js] Created: pure validation of warehouse definitions (including full topological build) and scenario parameters (bounds, known strategy, unknown keys rejected)
- [server/schemas.js] Created: shared Fastify schemas (route id coercion, run list filters)
- [server/routes/warehouses.js] Created: CRUD + JSON import/export for warehouses
- [server/routes/scenarios.js] Created: CRUD + JSON import/export for scenarios
- [server/routes/runs.js] Created: POST executes a simulation and persists it, GET list with filters, GET detail with traffic, DELETE
- [server/index.js] Created: bootstrap (wait DB → migrations → seed → listen) with graceful shutdown
- [web/public/index.html] Created: full-screen 3D page with left side panel — SimSteps header, transport (play/pause, x1/x10/x60, simulated clock, playback progress bar), scenario select, sliders (operators, B2C mix, order rate), spaghetti/heatmap switches, live KPI dashboard, save-run button, comparison section; import map for Three.js, inline SVG favicon, operator-state legend
- [web/public/js/layout.js] Created: pure geometry module (floor size, rack boxes, zone patches, aisle labels, slot count) computed from a warehouse definition, testable under Node
- [web/public/js/scene.js] Created: Three.js scene builder — floor + metric grid, rack volumes with edges and shadows, colored zone patches, text sprites, hemisphere + directional lighting, OrbitControls with damping
- [web/public/js/labels.js] Created: canvas-texture text sprites for 3D labels
- [web/public/js/main.js] Created: full orchestration — loads warehouse + scenarios from the API, runs the simulation in the browser (recorder + KPI sampler), rebuilds 3D layers on every parameter change, drives the playback clock (play/pause, x1/x10/x60, restart at end), refreshes live KPIs at 4 Hz, saves runs to the DB, comparison across current settings / scenarios / saved runs
- [web/public/js/timeline.js] Created: pure trajectory recorder plugged into engine hooks (onTravel/onState) and playback functions positionAt (constant-speed interpolation along paths) and stateAt
- [web/public/js/operators.js] Created: operator capsules colored by state (moving/picking/dropping/idle) with emissive glow, per-frame update from the timeline, dispose()
- [web/public/js/kpiSampler.js] Created: pure live-KPI computation (in-progress missions counted in occupancy), periodic sampling via the onEvent hook, kpiAt playback lookup
