# Changelog - 2026-07-05

## Summary
**Corridor network**: corridors become first-class objects (list, minimum 1) with editable position, length, lane width and orientation (horizontal/vertical dropdown); crossing or touching corridors connect automatically, forming real circulation paths between zones. The graph builder rewires accordingly: coordinate-keyed station nodes at endpoints/intersections, aisle ends connect to the nearest crossing horizontal corridor (at least one outlet required, dead-end aisles allowed), facilities attach by projection onto the nearest corridor, and a BFS connectivity check rejects broken networks with precise French errors. The historical `{frontY, backY}` format stays accepted everywhere and is normalized to two full-width corridors when editing. The engine is untouched (A* on the graph).


Display option: a « Masquer les libellés » switch in the Piloter tab hides all 3D label sprites (aisles, zones, corridors); while hidden, clicking an element — corridor bands included — reveals its own label (click in the void hides it again).

Editor follow-up on floor resizing: changing the warehouse width/depth now recenters the camera on the whole new floor (the extra depth grows toward the viewer and was off-screen, making the new space unreachable by drag), with a floor-proportional framing that keeps every corner visible. The front/back corridors — which bound aisle movement and were invisible (reported as "objects can't move along the whole y axis") — are now drawn on the floor as translucent circulation bands.

## Changes (network editing fixes)

- [web/public/js/editor/model.js] updateCorridor: an orientation flip now pivots the segment around its centre (it used to pivot around its start point, teleporting the far end), and any geometry change (orientation, length, width) re-clamps the segment inside the floor — a flip near an edge no longer produces an out-of-floor segment whose render silently lagged behind the model

## Notes (network editing fixes)
Both user-reported symptoms shared one root cause: applyWorkingDef only rebuilt the scene for fully valid definitions, so a model change that broke the network (orientation flip overflowing the floor, drag creating an island that captured the nearest zone) left the visuals on the last valid state — the flip "waited for the next move" and dragged corridors "kept coming back to the first magnet position". 176 unit + 16 UI tests pass.
