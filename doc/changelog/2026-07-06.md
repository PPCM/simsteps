# Changelog - 2026-07-06

## Summary
Edit-mode UX overhaul (V6 « poste de travail classique » from the mockup board): the floating bottom window — which had grown to 760 × 766 px and covered the scene — is replaced by a fixed desktop-application chrome inspired by Fusion 360 / Photoshop / LibreOffice: creation ribbon on top, right dock with a layers-style Structure tree above the selection properties, and a status bar (pointer floor coordinates, snap step, live validation state). New « Dupliquer » ribbon action backed by a pure `duplicateElement` model op.

## Changes

- [web/public/index.html] `#editPanel` floating window replaced by `#editChrome`: full-screen flex column with `pointer-events: none` in the middle (the scene stays clickable/draggable everywhere outside the panels); title bar (warehouse name, « Quitter sans enregistrer », « Enregistrer »), ribbon with captioned groups (Structure : Allée, Couloir · Zones : Atelier, Expédition, Réception, Parking, Tampon · Flux : Obstacle, Convoyeur · Modifier : Dupliquer, Supprimer) with tooltips, right dock (Structure tree + Propriétés + validation errors), bottom status bar (`#editCoords`, « Accrochage : 1 m », `#editValidity`, gesture hints); amber editing dot on the Configurer tab removed (the window is hidden while editing); `.legend[hidden]`/`.hint[hidden]` explicit display rules
- [web/public/js/editor/tree.js] Created (pure): `buildTree(def)` — typed groups with per-element summaries (aisle: bays · levels, corridor: orientation · length, zones: width × depth, conveyor: length · throughput), optional groups (parkings, buffers, obstacles, conveyors) omitted while empty
- [web/public/js/editor/panel.js] `renderTree`: collapsible groups (native details/summary) with per-group open state surviving re-renders, monospace rows with summary, selected row highlighted, click selects
- [web/public/js/editor/model.js] `duplicateElement(def, type, id)` (pure): clone under a fresh id (nextId per-type prefix) + « (copie) » label, offset 2 m through the existing move* clamps/snaps (corridors shift parallel to their axis), racks cloned with fresh ids for aisles
- [web/public/js/editor/controls.js] Optional `onHover` callback: floor-plane point under the pointer (throttled with the existing hover raycast, also fed during drags), null off-floor
- [web/public/js/main.js] Edit mode hides the floating windows, hint and legend and shows the chrome; Structure tree wired to selection (both directions); properties dock shows the selection or — with no selection — the warehouse globals; live validity in the status bar; Dupliquer handler (copy becomes the selection); `editWindow`/`editLocked`/`HINT_EDIT`/`editDot` removed (windows are hidden, not disabled)
- [tests/unit/web/tree.test.js] Created: mandatory groups with full counts, optional groups appearing when populated, summary formats
- [tests/unit/web/editor-model.test.js] duplicateElement: aisle with racks under unique ids, zone labelled « (copie) » with a valid plan, corridor shifted parallel to its axis, unknown type/id rejected in French
- [tests/ui/editeur.spec.js] First test asserts the chrome replaces the windows (ribbon, dock, status bar, title, validity) instead of disabled controls; corridor drag test moved to the BACK corridor at mid-length (line of sight clear of racks — the front corridor pick crossed the rack field — and no window to collapse anymore); aisle drag test updated: the old « stops at the corridor » assertion only held because the removed window swallowed the pointer mid-drag — the model clamps to the floor edge and the network stays valid via the front outlet; orbit drag point moved out of the ribbon area; new tests: tree selection + ribbon duplication persisted with racks, status-bar validity turning red on a bad bays value and floor coordinates fed on hover
- [README.md] Edit-mode section rewritten around the new chrome (ribbon, dock with tree, status bar)
