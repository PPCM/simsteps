# Changelog - 2026-07-06

## Summary
Edit-mode UX overhaul (V6 « poste de travail classique » from the mockup board): the floating bottom window — which had grown to 760 × 766 px and covered the scene — is replaced by a fixed desktop-application chrome inspired by Fusion 360 / Photoshop / LibreOffice: creation ribbon on top, right dock with a layers-style Structure tree above the selection properties, and a status bar (pointer floor coordinates, snap step, live validation state). New « Dupliquer » ribbon action backed by a pure `duplicateElement` model op.

## Changes

## Changes (undo/redo and keyboard shortcuts)

- [web/public/js/editor/history.js] Created (pure): bounded past/future stacks for immutable working definitions (model ops never mutate, states stored by reference); `push` clears the future, `undo`/`redo` swap the current state across the stacks
- [web/public/js/main.js] `applyWorkingDef` records the left state unless `{record: false}` (history application); ↶ Annuler / ↷ Rétablir buttons wired with live disabled state, selection dropped when its element no longer exists in the applied state; edit-mode keyboard shortcuts: 1–9 create elements in ribbon order, Ctrl+Z / Ctrl+Y (or Ctrl+Shift+Z) undo/redo, Ctrl+D duplicate, Suppr delete, Ctrl+S save, Échap deselect — inactive while typing in a dock field
